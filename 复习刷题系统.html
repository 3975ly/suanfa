📝 参考答案与解析：

正确排序：
O(2) < O(log n) < O(n2/3) < O(20n) < O(4n²) < O(3ⁿ) < O(n!)

详细解析：

① O(2) = O(1) — 常数级，与n无关
② O(log n) — 对数级，增长极慢（如二分搜索）
③ O(n2/3) — 亚线性，比线性慢，但快于n
④ O(20n) = O(n) — 线性，常数系数20不影响阶
⑤ O(4n²) = O(n²) — 平方级，常数系数4不影响阶
⑥ O(3ⁿ) — 指数级，增长极快
⑦ O(n!) — 阶乘级，增长最快

记忆口诀：常对幂线方指阶
O(1) < O(log n) < O(nc) < O(n) < O(n²) < O(aⁿ) < O(n!)

关键点：大O记号忽略常数系数和低阶项，只关注最高阶的增长趋势。
12
什么是算法？算法的特征有哪些？
📝 参考答案与解析：

一、算法的定义：
算法是指为解决某一特定问题而规定的一个有限长的操作序列。通俗地说，就是解决问题的方法或过程。

二、算法的五大特征：

① 输入（Input）：有零个或多个输入。零个输入的例子：输出一个固定值。
② 输出（Output）：有一个或多个输出。算法必须有结果输出，否则无意义。
③ 确定性（Definiteness）：每一步操作都是确切定义的，无二义性。相同输入必产生相同结果。
④ 有穷性（Finiteness）：算法必须在有限步之后终止。注意：程序可以不满足有穷性（如操作系统），但算法必须终止。
⑤ 可行性（Effectiveness）：每一步都可以通过基本运算执行有限次来实现。

注意：算法≠程序。程序不必满足有穷性，算法必须。

给出算法的定义？何谓算法的复杂性？计算以下代码最坏情况下的时间复杂性。
for(j=1;j<=n;j++)
  for(i=1;i<=n;i++)
    {c[i][j]=0;
     for(k=1;k<=n;k++)
       c[i][j]=c[i][j]+a[i][k]*b[k][j];}
📝 参考答案与解析：

一、算法的定义：
指在解决问题时，按照某种机械步骤一定可以得到问题结果的处理过程。

二、算法的复杂性：
指算法在运行过程中所需要的资源（时间、空间）的多少。
• 所需资源越多 → 复杂性越高
• 通常用输入规模n的函数来表示
• 主要关注最坏情况下的复杂性

三、时间复杂度计算过程：
分析三重嵌套循环：
• 第1层（j循环）：j从1到n，执行 n 次
• 第2层（i循环）：i从1到n，每次执行 n 次
• 第3层（k循环）：k从1到n，每次执行 n 次
• 基本操作 c[i][j]=c[i][j]+a[i][k]*b[k][j]
  共执行 n × n × n = n³ 次

∴ T(n) = O(n³)

这是标准矩阵乘法算法。Strassen算法可优化到O(n2.81)。

算法A和算法B解同一问题，若要使算法A时间复杂性的阶高于算法B，a的最大整数值可取多少？
📝 参考答案与解析：

题目条件：
算法A的时间复杂性满足：T(n) = aT(n/2) + O(n)（a为待定常数）
算法B的时间复杂性满足：T(n) = 2T(n/2) + O(n log n)

求解过程：
① 由Master定理解算法A的递归方程：
  T(n) = aT(n/2) + O(n)
  比较 nlog₂a 与 n¹ 的大小：
  • 若 a < 2：T(n) = O(n)
  • 若 a = 2：T(n) = O(n log n)
  • 若 a > 2：T(n) = O(nlog₂a)

② 解算法B的递归方程：
  T(n) = 2T(n/2) + O(n log n)
  解得：T(n) = O(n log²n)

③ 要使A的阶 > B的阶，即 nlog₂a > n log²n：
  需要 log₂a > 1（当n足够大时多项式增长超过对数增长）
  但更精确地：需要 nlog₂a > n·log²n
  即 log₂a > 1，故 a > 2（此时A的阶高于B）
  当 a ≤ 16 时，O(nlog₂16) = O(n⁴) > O(n log²n) ✓
  当 a = 16 时 log₂16 = 4，A的阶 = O(n⁴)

④ 最终答案：a的最大整数值为 15
（当a=16时阶恰好相等，不满足高于的严格要求）

算法分析的目的是什么？
📝 参考答案与解析：

算法分析的目的：

1）性能预估：对算法的某些特定输入，估算该算法所需的内存空间和运行时间，在实际运行前了解其资源消耗。

2）比较与选择：建立衡量算法优劣的标准，用以比较解决同一问题的不同算法，从中选择更高效的方案。

具体来说：
• 通过时间复杂度分析，判断算法是否能在合理时间内处理大规模数据
• 通过空间复杂度分析，判断算法是否会超出内存限制
• 通过最坏/平均/最好情况分析，全面了解算法的性能表现
• 为实际工程中的算法选型提供理论依据

算法设计常用的技术有哪些？（写5种以上）
📝 参考答案与解析：

算法设计常用技术（7种）：

①分治法：将问题分解为规模较小的子问题，递归求解后合并（如归并排序、快排）
②回溯法：在解空间树中DFS搜索，不满足约束则回溯（如N皇后、图着色）
③贪心法：每步做局部最优选择，期望达到全局最优（如活动安排、Dijkstra）
④动态规划法：保存子问题解避免重复计算，自底向上求解（如LCS、背包）
⑤分支限界法：BFS搜索解空间，用界函数剪枝求最优解（如0-1背包）
⑥蛮力法：穷举所有可能解逐一验证（如百鸡百钱）
⑦倒推法：从已知结果逆向推导初始条件（如穿越沙漠问题）

选择建议：问题有最优子结构且子问题重叠→DP；子问题独立→分治；贪心选择性成立→贪心；需要搜索所有解→回溯/分支限界。

什么是递归算法？递归算法的特点？
📝 参考答案与解析：

一、递归算法定义：
一个模块（函数/过程）除了可以调用其它模块外，还可以直接或间接地调用自身的算法。
• 直接递归：函数A调用A自己
• 间接递归：A调用B，B再调用A

二、递归算法两大特点：

① 必须有递归终止条件（边界条件）
  没有终止条件 → 无限递归 → 栈溢出崩溃
  例：factorial(0)=1 就是终止条件

② 递归方程（递归步骤）
  用较小规模的自身来定义较大规模的问题
  例：factorial(n)=n×factorial(n-1)
  每次调用都使问题规模缩小，最终到达边界

三、递归的本质：
系统用栈保存每层递归的局部变量和返回地址。递归深度过大会栈溢出。

经典递归：阶乘、Fibonacci、汉诺塔、二分搜索、归并排序、树遍历
27
比较循环与递归的异同？
📝 参考答案与解析：

一、相同点：
递归与循环都是实现重复操作的机制，都能解决需要反复执行步骤的问题。

二、不同点：

① 效率：
  递归：额外开销大——函数调用/返回需时间，栈帧需空间
  循环：无额外调用开销，效率更高
  例：递归fib(40)需指数时间，循环O(n)即可

② 可转换性：
  循环 → 总可以转为递归 ✓
  递归 → 不一定能简单转为循环（需手动用栈模拟）
  例：树的递归遍历转循环需显式栈

③ 灵活性：
  循环：嵌套层数编译时固定
  递归：深度运行时动态决定，更灵活
  例：遍历不确定深度的树结构，递归更自然

选择建议：有自然递归结构（树/分治）→递归；追求效率→循环或尾递归优化。

递归算法解题通常有哪三个步骤？
📝 参考答案与解析：

递归算法解题三步骤：

第一步：分析问题、寻找递归
找出大规模问题与小规模问题的递推关系。
例：汉诺塔n个盘 = 移n-1个到辅助柱 + 移第n个 + 移n-1个到目标柱
关键：确保每次递归使问题规模严格缩小

第二步：设置边界、控制递归
找出停止条件（最小规模问题的直接解）。
例：汉诺塔n=1时直接移动；阶乘n=0时返回1
关键：边界条件必须能被达到，否则无限递归

第三步：设计函数、确定参数
确定函数签名——需要哪些参数描述问题状态。
例：hanoi(n, from, to, aux) 需要盘数和三个柱子
关键：参数要能完整描述当前子问题

验证：①确认边界正确 ②假设子问题正确则当前层正确（归纳法）

适合用分治算法求解的问题具有哪些基本特征？
📝 参考答案与解析：

适合分治的四个特征：

① 规模缩小到一定程度就容易解决
  绝大多数问题满足——n足够小时可直接求解
  例：1个元素的数组天然有序

② 可分解为若干规模较小的相同问题
  原问题与子问题结构相同，只是规模不同
  例：排序n个数 = 排序两个n/2子数组 + 合并

③ 子问题相互独立（分治与DP的关键区别！）
  各子问题无公共子问题，不会重复计算
  例：归并排序左右两半互不影响 ✓
  反例：Fibonacci有大量重叠子问题 → 应用DP

④ 子问题的解可合并为原问题的解
  如果无法有效合并就不能用分治
  例：归并排序O(n)时间合并两个有序数组

经典分治：归并排序、快排、二分搜索、棋盘覆盖、Strassen矩阵乘法
30
分治算法的基本思想和解题步骤？
📝 参考答案与解析：

一、分治算法基本思想：
将一个难以直接解决的大问题，分割成若干个规模较小的相同子问题，各个击破，分而治之。

二、解题步骤：

① 分解（Divide）：
  将原问题分解为若干规模较小、相互独立、与原问题形式相同的子问题
  例：归并排序将数组从中间一分为二

② 求解（Conquer）：
  递归求解各子问题。若规模足够小则直接求解（递归出口）
  例：子数组只有1个元素时天然有序

③ 合并（Combine）：
  将子问题的解合并为原问题的解
  例：将两个有序子数组合并为一个有序数组

三、复杂度通式：T(n) = aT(n/b) + f(n)
  a=子问题个数，n/b=子问题规模，f(n)=分解合并代价
  可用Master定理求解

楼梯有n个台阶，每次可上1步或2步，求上楼方法数F(n)。写出递归表达式和算法。
📝 参考答案与解析：

一、问题分析：
到达第n阶，最后一步只有两种走法：

动态规划算法基本思想？
📝 参考答案与解析：

动态规划基本思想：

与分治法类似，将待求解问题分解成若干子问题。

关键区别：分解得到的子问题往往不是互相独立的——不同子问题有公共的更小子问题（重叠子问题）。

① 用一个表（数组）保存已解决子问题的答案
② 需要时直接查表而非重新计算
③ 自底向上按规模从小到大依次求解

这样就把指数级时间降为多项式时间。

适用条件：最优子结构 + 子问题重叠

📝 参考答案与解析：

一、问题描述：

二、最优子结构分析：
设c[i][j]为Xi=(x1..xi)和Yj=(y1..yj)的LCS长度：
• 若xi=yj → 这个字符一定在LCS中，c[i][j]=c[i-1][j-1]+1
• 若xi≠yj → LCS要么不含xi要么不含yj，取较大者

三、递归公式：
c[i][j] = 0                          (i=0 或 j=0)
c[i][j] = c[i-1][j-1] + 1            (xi = yj)
c[i][j] = max(c[i-1][j], c[i][j-1])  (xi ≠ yj)
四、算法：
自底向上填表（i从1到m，j从1到n），同时记录方向标记b[i][j]用于回溯：
for(i=1;i<=m;i++)
  for(j=1;j<=n;j++){
    if(x[i]==y[j]){
      c[i][j]=c[i-1][j-1]+1; b[i][j]="↖";
    }else if(c[i-1][j]>=c[i][j-1]){
      c[i][j]=c[i-1][j]; b[i][j]="↑";
    }else{
      c[i][j]=c[i][j-1]; b[i][j]="←";
    }
  }
五、构造最优解：从b[m][n]开始回溯，遇到"↖"输出该字符。

六、时间复杂度：O(mn)，空间O(mn)（可优化为O(min(m,n))）

      j = i;  // 更新最后选入的活动
    } else {
      A[i] = false;
    }
  }
}

一、Prim算法：
思想：从任一顶点出发逐步"生长"最小生成树。
步骤：
① 任选一个顶点加入集合S
② 从S到V-S的所有边中选权最小的边(u,v)
③ 将v加入S，边(u,v)加入树
④ 重复直到S包含所有顶点（n-1条边）
复杂度：O(n²)（邻接矩阵），O(e·log n)（堆优化）

二、Kruskal算法：
思想：按边权从小到大选边，不形成环则加入。
步骤：
① 所有边按权值升序排序
② 依次考察每条边：加入后不形成环则选入
③ 直到选了n-1条边
④ 判环：并查集（两端点是否同一集合）

Dijkstra算法计算单源最短路径。
📝 参考答案与解析：

一、问题：求从源顶点到所有其他顶点的最短路径（单源最短路径）。

二、算法思想：
维护集合S（已确定最短路径的顶点）。每次从V-S中选距源最近的顶点u加入S，并用u松弛相邻顶点。

三、详细步骤：
① 初始化：dist[源]=0，dist[其余]=∞，S=∅
② 从V-S中选dist值最小的顶点u，加入S
③ 松弛：对u的每个邻接顶点v：
  if dist[u]+w(u,v) < dist[v]
    dist[v] = dist[u]+w(u,v)
④ 重复②③直到S包含所有顶点

四、代码：
void Dijkstra(int s){
  dist[s]=0; 其余dist[]=∞;
  for(i=1;i<=n;i++){
    u = dist最小的未访问顶点;
    visited[u] = true;
    for(u的每个邻接顶点v)
      if(dist[u]+w[u][v]<dist[v])
        dist[v]=dist[u]+w[u][v];
  }
}
五、复杂度：O(n²)。堆优化后O((n+e)log n)。

六、注意：Dijkstra不适用于负权边（用Bellman-Ford）。贪心正确性基于：已加入S的顶点其dist值不会再变小。

📝 参考答案与解析：

一、回溯算法基本思想：
在问题的解空间树中，按深度优先策略从根节点出发搜索。

• 满足 → 继续向子节点深入
• 不满足 → 剪枝，回溯到父节点，尝试其它分支
• 到达叶子节点 → 得到一个可行解

① 定义问题的解空间
  明确解的形式，如x=(x1,x2,...,xn)，每个xi的取值范围

② 确定解空间的组织结构（树的类型）
  子集树（选/不选）、排列树（全排列）、满m叉树（m种选择）

③ 以DFS搜索解空间，用剪枝函数提高效率
  约束函数：剪去不满足约束的子树
  限界函数：剪去不可能产生最优解的子树

回溯法 = DFS + 剪枝，没有剪枝就退化为暴力搜索。

简述分支限界法的基本思想与解题步骤？
📝 参考答案与解析：

一、基本思想：
以广度优先(BFS)或最小耗费优先的方式搜索解空间树。

与回溯法的关键区别：
• 每个活结点只有一次机会成为扩展结点
• 扩展时一次性生成所有子结点
• 用限界函数对子结点估值，不满足条件的直接丢弃

① 定义解空间：确定解的形式和约束条件

② 组织搜索：以BFS方式逐层扩展，或按优先级选最有希望的结点

③ 剪枝：
  • 约束函数：判断可行性（是否违反约束）
  • 限界函数：估算当前结点能达到的最优值上/下界
  • 若上界 ≤ 当前已知最优解 → 剪枝

④ 更新最优解：到达叶子结点时，若优于当前最优则更新

三、两种实现：
• 队列式(FIFO)：简单，按层序扩展
• 优先队列式：更高效，优先扩展最有希望的结点

概率算法是指在算法执行过程中，某些步骤的选择依赖于随机数，即允许算法随机选择下一个计算步骤。

概率算法的基本特征：

对所求解问题的同一实例，用同一概率算法求解两次，可能得到完全不同的效果。

具体表现：
• 运行时间可能不同（有时快有时慢）

• 期望时间复杂度较低
• 错误概率可控（多次运行可降低）

一、易解问题：
存在多项式时间算法的问题（O(nᵏ)，k为常数）。
例：排序O(n log n)、最短路径O(n²)、MST O(e log e)

目前只知道需要指数时间（如O(2ⁿ)、O(n!)）才能解决的问题。

三、难解问题的两类：

① 不可判定问题：
  根本不存在任何算法能求解。
  经典例子：停机问题（Halting Problem）——无法判断任意程序是否会终止。
  这类问题"太难了"，超越了计算的极限。

② 非决定的难处理问题：
  问题是可判定的（可解），但即使使用非确定性计算机也不能在多项式时间内求解。
  目前没有找到多项式算法，也没有证明不存在。
  NP完全问题就属于这一类（如果P≠NP的话）。

什么是P类问题？NP问题？NP完全问题？
📝 参考答案与解析：

一、P类问题：
能用确定性算法在多项式时间内求解的判定问题。
• "确定性"：每步只有一种选择
• 所有易解问题都属于P类
• 例：排序、最短路径、MST、二分搜索

二、NP类问题：
给定一个解（证书），能在多项式时间内验证其是否正确的问题。
• NP = Non-deterministic Polynomial
• P ⊆ NP（能求解的一定能验证）
• 核心问题：P = NP ? （千禧年七大难题之一）
• 例：给一个TSP路径，O(n)即可验证其长度是否≤k

三、NP完全问题(NPC)：
满足两个条件：
① 属于NP（解可以多项式验证）
② NP中所有问题都可以多项式时间归约到它（NP-hard）

意义：若任何一个NPC问题找到多项式算法 → P=NP → 所有NP问题都可多项式求解。
目前普遍猜测P≠NP，即NPC问题没有多项式算法。

NP完全问题的解题策略：

由于NPC问题目前无多项式精确算法，实际中采用以下策略：

① 只对问题的特殊实例求解
  某些特殊结构的实例可以高效求解
  例：平面图着色可在O(n)内4-着色

② 用动态规划或分支限界法
  虽然最坏仍是指数级，但通过剪枝在实际中可能快速求解
  例：0-1背包用DP可O(nW)求解（伪多项式）

③ 启发式方法
  基于经验或直觉的搜索策略，不保证最优但实际效果好
  例：遗传算法、模拟退火、蚁群算法

④ 只求近似解
  在多项式时间内求一个"足够好"的近似解
  用近似比衡量解的质量
  例：贪心法求TSP可保证不超过最优解的2倍

近似算法应满足哪两个基本要求？
📝 参考答案与解析：

近似算法的两个基本要求：

① 时间复杂性：
  算法必须能在n的多项式时间内完成。
  如果不要求多项式时间，直接用指数级精确算法即可。
  近似算法的价值在于：用多项式时间换取近似解。

② 解的近似程度：
  近似解应满足一定的精度保证。

  衡量标准：
  • 近似比 ρ(n)：
    max(C/C*, C*/C) ≤ ρ(n)
    其中C为近似解值，C*为最优解值
    ρ(n)越接近1越好

  • 相对误差 ε：
    |C-C*|/C* ≤ ε
    ε越小近似越精确

理想目标：在O(多项式)时间内找到近似比为常数（甚至1+ε）的解。
